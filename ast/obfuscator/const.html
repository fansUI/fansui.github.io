# 对象属性的两种访问方式

对象.属性名;
对象.方法名();
对象["属性名"];
对象["方法名"]();

# 修改（重构）对象属性访问方式

1.把对象属性和对象方法的访问方式全部改成中括号方式
2.把标准内置对象的调用放到 window 对象中

  • 为什么要修改访问方式？

因为把对象属性或对象方法转换成中括号的访问方式，属性名或者方法名就变成了字符串（为混淆加密做准备），针对字符串我们可以对其进行拆分、拼接、编码、加密等操作。

function Person(name) {
  this.name = name;
}

let p = new Person("张三");

// 点语法
console.log(p.name);
// 中括号语法
console.log(p["name"]);

let a = "na";
let b = "me";
console.log(p[a + b]);

// 把标准内置对象Date转换成window['Date']
// 这样就把Date转成了字符串
new Date();
new window["Date"]();

# 字符串转 ASCII 码转十六进制（只有字符串才可以）

改变对象属性的访问方式之后，代码的阅读性仍然较高，需要集训进行复杂化处理。
因为在 js 中的字符串支持十六进制形式表示，所以可以用字符串的十六进制形式来代替原有的字符串。
先把字符转成 ACSII 码，再转成十六进制

function hexEnc(code) {
  for (var hexStr = [], i = 0; i < code.length; i++) {
    s = code.charCodeAt(i).toString(16);
    hexStr += "\\x" + s;
  }
  return hexStr;
}

# 字符串的 ASCII 码混淆

  • 字符串可以直接进行 ASCII 码混淆
// 字符转ASCII码
"d".charCodeAt(0);
// ASCII码转字符串
String.fromCharCode(100);


String.fromCharCode === ''['constructor']  // true
  • 代码语句也可以用 ACSII 码进行混淆 但这时 ASCII 再转回代码不能执行了，js 中有 2 中方式把字符串当成代码执行（eval 和 new Function），微信小游戏禁用了这 2 个方法，因此不能热加载了，但是针对简单的小游戏来说可以使用第三方插件 Sval

# 字符串的常量加密（base64、其他方式）

字符串常量加密的核心思想是：先把字符串加密得到密文，然后在使用之前，调用对应的解密函数去解密，得到明文。
代码中仅仅出现解密函数和密文。当然，也可以使用不同的加密方法去加密字符串，然后再调用不同的解密函数去解密。

# 数值常量加密

利用位异或的特性来加密

a ^ b = c;
c ^ b = a;
// a相当于明文
// b相当于秘钥
// c相当于密文
