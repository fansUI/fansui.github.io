# 什么是程序

  • 硬件以外的都是程序，就是一堆代码文件
  • 硬件以外的都是软件，分为操作系统和应用程序，就是一堆代码

计算机就是一堆废铁，计算机的运行全部受程序控制，程序是计算机的灵魂 计算机受电支配，高低电流，代表 0101010101，计算机只认识 0 和 1，早起编程是电流控制， 计算机一堆废铁----->操作系统的出现，操作硬件，提供系统调用接口 api---->终端 shell 解释器----->高级应用程序

软件分为 2 类：

  • 操作系统（计算机硬件的控制程序）：就是一个协调、管理、控制计算机硬件资源与高级软件资源（应用程序）的控制程序
  • 应用程序（应用软件）：借助操作系统或操作系统提供的调用接口来操作计算机硬件

计算机就是一堆废铁，首先应该让硬盘转起来（提供一个接口），显示器亮起来（提供一个接口），扬声器响起来（提供一个接口），操作系统就是用来操作硬件 其次应用程序借助操作系统提供的接口来操作硬件

# 制作视频播放器（高级功能，高级应用程序），相当于借助了操作系统封装的系统调用接口，

  • 需要调用硬盘读取接口（因为视频数据存储在硬盘中）
  • 需要调用显示器控制接口（因为视频需要显示在显示器中）
  • 需要调用扬声器（用来播放声音）
  • 需要调用内存...
  • 需要调用 cpu...

没有操作系统的时代，每家公司都要写硬件控制程序（如果你能发现行业中很多人都在做重复工作，你的商机来了，），因此操作系统出现了（dos、）

# 计算机体系的三层结构

  • 计算机硬件
  • 操作系统
  • 应用程序

操作系统和应用程序最终都是操作硬件，只不过应用程序需要借助操作系统来操作硬件

# 买电脑

  • 买来电脑如果没有安装操作系统（也就是裸机）只是一堆废铁，只能通过电流控制，一般人做不到
  • 因此为了方便人们使用，操作系统出现（只有内核），假设有十万功能，提供了十万接口（比之前简单），但是使用也很麻烦，因此操作系统有提供了系统接口，对十万功能接口封装成一万接口，但是还是很难很复杂
  • 因此很多人又根据操作系统提供的系统接口，再次进行封装，从而出现了 shell 终端解释器（应用程序）
  • shell 终端依然很难被大众使用，因此有人借助 shell 来封装高级应用程序（解释型语言开发的应用程序）（图形界面，鼠标键盘操作）
  • 还有部分人直接使用操作系统提供的系统接口来封装高级应用程序（图形界面，鼠标键盘操作），因为少了 shell 终端解释器这一层，因此执行效率（运行效率）相对较高

用户点点鼠标--->被 python 应用程序捕捉--->传递给 python 解释器--->操作系统内核接口--->操作控制硬件

# 计算机系统 5 大组成部分

  • 控制器

  • 运算器

    控制器和运算器集成在一个硬件（cpu）之上，又称为中央处理器

  • 存储器：用来存取数据

    • 内存：比如内存条
      • 基于点存取数据，断电数据全部丢失
      • 用于临时保存数据，存取速度都快
    • 外存：比如磁盘
      • 基于磁存取数据，断电数据仍然存在
      • 用于永久（理论上永久）保存数据，存取速度都慢

  • 输入设备

    • 键盘
    • 鼠标

  • 输出设备

    • 显示器
    • 扬声器

# 程序的运行

  • 程序最先是存放于硬盘中的

  • 程序的运行需要先经理加载的过程：程序的代码或者数据从硬盘读入到内存

  • 然后 cpu 再从内存中读取指令来运行

    ps：安装软件都安装到硬盘中 ps：软件的运行，代码与数据都加载到了内存中 ps：因此电脑安装软件的个数取决于硬盘 ps：计算机同时运行软件的数量取决于内存的大小，启动的软件多了的话，电脑会卡，加内存条 ps：内存一断电就没了，因此软件不可能安装到内存中 ps：计算机所有硬件的运行都受 cpu 指挥，但是 cpu 也是硬件，受谁指挥？ ps：只要是硬件都是软件来支配的，人类的程序指令来控制 cpu，cpu 把程序指令转化为自身的指令来控制其他硬件

# 发展史

  • 第一代计算机（1940-1955）：真空管（容易烧坏）和穿孔卡片

计算机资源浪费，预约时间

  • 第二代计算机（1955-1965）：晶体管和批处理系统（批处理：一批一批处理，批量处理）

节省了计算机资源，但浪费了人力 此时的批处理，一次只把一个程序读入内存，运行完毕后，在执行下一个程序，没有并发，称之为串行

  • 第三代计算机（1965-1980）：集成电路芯片和多道程序设计（多道技术）

# 进程

  • 操作系统最核心的概念：进程（进程指的就是一个程序的运行过程，是由操作系统来控制硬件来运行的）
  • 程序的运行过程

程序从硬盘加载到内存，cpu去内存中取指令来操作硬件运行，程序运行过程可能会产生数据，数据可能需要写入硬盘中
