# 一：谈谈你对闭包的理解

# 0.回答的方向

  1. 先回答闭包是什么
  2. 闭包怎么样
  3. 闭包怎么用
  4. 自己平时在公司里是怎么用的

# 1.闭包是什么

闭包，在 javascript 高级程序设计中定义如下：

闭包是有权访问其他函数作用域的局部变量的一个函数

当然，我们还需要做到补充以下关于闭包的理解

在原生 js 中，函数中变量的作用域仅属于它所在的函数作用域，在函数执行完后将会被回收机制清理，内存也会被收回，但是我们如果在里面添加了一个闭包，那么通过闭包可以访问上级作用域的原因，即使上级函数执行完，作用域也不会销毁，在此处的子函数便是闭包，此时它具有了范文上级作用域的权限，即使上级函数执行完，上级函数的作用域也不会销毁。

# 2.闭包怎么样

在此处，我们应该从闭包解决了哪些问题，有什么有点和缺点来回答，

在本质上，闭包就是将函数内部和外部连接起来的一座桥梁。

当然，我们也进行自己的补充

优点：闭包可以缓存上级作用域，因此就使得函数外部打破了“函数作用域”束缚，可以访问函数内部的变量。在之前项目中我们使用的 AJAX 请求成功后的回调函数就是闭包的应用，它正是利用了闭包的上述访问上级作用域的特性，在回调函数中就拥有了它的上级函数作用域的访问和操作能力，因此，我们便不需要使用钩子函数去操作上级函数作用域内部的变量了

一句话总结：闭包两个作用，一是读取函数内部的变量，二是让这些变量的值始终存在内存之中。

# 二：普通函数、构造函数、构造函数内部方法中的 this 分别指向？

# 1.普通函数

对于普通函数:由于没有对象的或者实例的调用,这就相当于在全局当中打印 this,所以 this 都是指向 js 的顶级对象 window.(箭头函数形式的普通函数也一样)

*注：若使用严格模式，则普通函数中 this 为 undefined，普通的箭头函数还是 window（实验环境为谷歌浏览器）

# 2.构造函数

构造函数中的 this 指向是：this 指向的是调用它的实例对象.

# 3.对象中的方法

对象中的普通方法指向对象（调用者），箭头函数的形式中 this 指向定义时所在上下文环境

var obj = {
    a: 'haha',
    b: this,
    c: function(){
        console.log(this.a)
    },
    d(){
        console.log(this.a)
    },
    e: () => {
        console.log(this.a)
    }
}
obj.c()     // haha
obj.d()     // haha
obj.e()     // undefined：箭头函数中this指向obj中的this，相当于obj.b的值，obj.b在这里代表window

# 四：call、apply 和 bind 的异同点

  • 相同点
  1. 三个方法都是用来修改 this 指向
  2. 三个方法的第一个参数都是 this 对象
  • 不同点
  1. call 和 apply 的区别只是参数的传递方式不同，call 可以传递多个参数（参数列表），apply 需要的是一个参数数组
  2. bind 方法返回一个新的改变了 this 指向的函数，不会立即执行，需要调用，call 和 apply 会立即执行
  3. bind 参数和 call 参数形式相同，都是参数列表

# 五：对象的深拷贝和浅拷贝

深复制和浅复制只针对像 Object, Array 这样的复杂对象的
浅复制只复制一层对象的属性，而深复制则递归复制了所有层级。 \

  1. 浅拷贝：拷贝出来的只是一个引用，指向同一个内存空间（内存地址）
  2. 深拷贝：拷贝出来的和原变量完全分离，是分别独立的数据
  • 浅拷贝
// 方式一：自定义函数
var obj = { a: 1, arr: [2, 3] };
var obj2 = copy(obj);
obj.arr[0] = 10;
console.log(obj2.arr[0]); // 10

function copy(src) {
  var dst = {};
  for (var prop in src) {
    if (src.hasOwnProperty(prop)) {
      dst[prop] = src[prop];
    }
  }
  return dst;
}

// 方式二：借助Object.assign()：多层是浅拷贝 一层是深拷贝
// 当对象只有一级属性为深拷贝;
// 当对象中有多级属性时，二级属性后就是浅拷贝;
let obj = { a: 1, b: [1, 2, 3] };
let obj2 = Object.assign({}, obj);

// 方式三：ES6 扩展运算符
let obj = { a: 1, b: [1, 2, 3] };
let obj2 = { ...obj };

# 七、v-if 和 v-show 的区别

  1. v-if 是惰性的，如果条件为假，则什么也不做，v-show 不管条件真假都会编译
  2. v-if 有更高的切换消耗；v-show 有更高的初始渲染消耗
  3. v-show 适合频繁切换，v-if 适合条件不怎么改变的情况
  4. v-if 是通过控制 dom 节点来实现，v-show 是通过样式来控制

# 八、methods 和 computed 的区别

  • 计算属性是基于它们的响应式依赖进行缓存的，只在相关响应式依赖发生改变时它们才会重新求值
  • 调用方法将总会再次执行函数

# 九、computed 和 watcher 的区别

  • computed 是有缓存的，只有其依赖的数据发生变化时才会重新执行，不支持异步操作
  • watcher 不支持缓存，支持异步和大开销的操作
  • 一个是计算，一个是监听，语义不同

# 十四、EventBus：事件总线

// event-bus.js
import Vue from "vue";
export const EventBus = new Vue();

// 发送事件
EventBus.$emit("aMsg", "来自A页面的消息");

// 接收事件
EventBus.$on("aMsg", (msg) => {
  // A发送来的消息
  this.msg = msg;
});

// 发送消息
EventBus.$emit(channel: string, callback(payload1,…))

// 监听接收消息
EventBus.$on(channel: string, callback(payload1,…))

你也可以使用EventBus.$off('aMsg')来移除应用内所有对此某个事件的监听。
或者直接调用EventBus.$off()来移除所有事件频道，不需要添加任何参数。
